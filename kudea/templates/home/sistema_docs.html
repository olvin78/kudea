1. Arquitectura del Sistema de IVA

1.1. Concepto Fundamental

En KUDEA, el precio de cada producto ya incluye el IVA. Esto significa que cuando un cajero escanea un producto y ve 12.10 €, ese es el precio final que paga el cliente.

✅ Ejemplo concreto (IVA 15%):
Un producto con precio 11.50 € (IVA incluido al 15%):
Base Imponible = 11.50 / 1.15 = 10.00 €
IVA = 11.50 - 10.00 = 1.50 €
Total = 11.50 € (el cliente paga exactamente lo que ve)

1.2. Flujo del IVA en el TPV

  1. Producto: Cada producto tiene su propio % de IVA (campo porcentaje_iva)
  2. Ticket: Cuando se añade un producto, el frontend guarda el precio y el % de IVA
  3. Cálculo: Se extrae el IVA del precio: IVA = precio - (precio / (1 + tasa_iva))
  4. Descuento: Se aplica proporcionalmente a cada línea antes de extraer el IVA
  5. Total: Es la suma de precios menos descuento (el IVA ya está incluido en los precios)
📌 Fórmula clave:
IVA = precioConIVA - (precioConIVA / (1 + porcentajeIVA / 100))

NO se hace: IVA = precioBase × porcentajeIVA / 100 (eso sumaría IVA a un precio que ya lo incluye)

2. Configuración del IVA

2.1. IVA General (Default)

En Configuraciones → Impuestos & IVA puedes definir el IVA general del sistema:

  • Campo numérico libre: Puedes escribir cualquier valor (15, 10, 0, 7, 20, etc.)
  • Valor por defecto: 15% (configurado para Nicaragua)
  • Se guarda en: ConfiguracionFiscal.iva_general y ConfiguracionTPV.iva_por_defecto

2.2. IVA por Producto

Cada producto tiene su propio IVA. Puedes cambiarlo:

  • Al crear/editar: Campo numérico donde escribes el % (ej: 15, 10, 0)
  • En lote: Usando el checkbox "Sincronizar e Imponer IVA Global" en Configuraciones
  • Masivamente: Desde el admin de Django (lista de productos editable)

2.3. Sincronización Masiva

⚡ Superpoder: En Configuraciones → Impuestos & IVA, marca "Sincronizar e Imponer IVA Global" y todos los productos se actualizarán al nuevo porcentaje.

3. Estructura de Datos

3.1. Modelo Producto

CampoTipoDescripción
precioDecimalPrecio de venta al público IVA incluido
porcentaje_ivaDecimal (0-100)% de IVA del producto (ej: 15.00)
descuentoDecimal (0-100)% de descuento sobre el precio final
stockIntegerUnidades disponibles en inventario
categoriaFKCategoría del producto (agrupación fiscal opcional)

3.2. Modelo Venta

CampoTipoDescripción
subtotalDecimalSuma de precios de todas las líneas (IVA incl.)
descuentoDecimalDescuento total aplicado en €
ivaDecimalTotal de IVA extraído (NO sumado al total)
totalDecimal= subtotal - descuento (el IVA ya está incluido en subtotal)
recibidoDecimalDinero entregado por el cliente
cambioDecimal= recibido - total (nunca negativo)

3.3. Modelo DetalleVenta

CampoDescripción
productoFK al producto vendido
cantidadUnidades vendidas
precio_unitarioPrecio en el momento de la venta
total= precio_unitario × cantidad
📌 Nota: El IVA no se almacena en DetalleVenta. Se obtiene del Producto.porcentaje_iva al consultar. Si cambias el IVA de un producto, los tickets históricos mostrarán el IVA nuevo (no el de la fecha de venta).

4. Flujo de una Venta

4.1. Paso a paso

  1. Apertura de caja: El cajero abre la caja con un PIN y fondo inicial
  2. Selección de productos: Busca y añade productos al ticket
  3. Validación de stock: No permite añadir más unidades de las disponibles
  4. Descuento: Opcional, se aplica proporcionalmente a todas las líneas
  5. Método de pago: Efectivo, tarjeta, bizum, pago mixto
  6. Guardar venta: El backend recalcula todo desde la BD, no confía en el frontend
  7. Stock: Se descuenta del inventario
  8. Cashflow: Se registra el movimiento contable
  9. Ticket: Se muestra el resumen con desglose de IVA

4.2. Seguridad del Cálculo

✅ El backend siempre recalcula:
El frontend envía [{id, precio, cantidad}]. El backend:
1. Vuelve a buscar cada producto de la BD
2. Usa el porcentaje_iva real de la BD
3. Recalcula IVA, descuento y total
4. Verifica stock suficiente
5. Verifica que recibido ≥ total
Si algo no cuadra, la venta se rechaza.

5. Cálculos Detallados

5.1. Ejemplo con IVA 15% (Nicaragua)

ProductoPrecioCant.TotalIVABase
Arroz 1kg11.50223.003.0020.00
Aceite 1L34.50134.504.5030.00
Leche 1L10.00330.003.9126.09
Subtotal87.5011.4176.09
Descuento (5%)-4.38
TOTAL83.12 €

Cálculo de cada línea:

  • Arroz: IVA = 23.00 - (23.00 / 1.15) = 23.00 - 20.00 = 3.00
  • Aceite: IVA = 34.50 - (34.50 / 1.15) = 34.50 - 30.00 = 4.50
  • Leche: IVA = 30.00 - (30.00 / 1.15) = 30.00 - 26.09 = 3.91
  • Descuento proporcional: tasa = 4.38 / 87.50 = 0.05 (5%)
  • Total = 87.50 - 4.38 = 83.12 €

5.2. Ejemplo con IVA 0% (Exento)

ProductoPrecioCant.TotalIVABase
Pan (0% IVA)5.00210.000.0010.00
Arroz (15% IVA)11.50111.501.5010.00
Total21.50 €
📌 Regla: IVA 0% → el precio es todo base imponible, no se extrae IVA.

6. Configuración Recomendada para Nicaragua

  • IVA General: 15% (estándar Nicaragua)
  • IVA 0%: Para productos exentos (canasta básica)
  • IVA 7%: Si se requiere para algún producto específico
  • Símbolo monetario: Configurable en Configuraciones TPV
  • PIN de apertura: 4 dígitos, configurable

7. Reportes y Arqueo

7.1. Desglose de IVA en Tickets

El ticket muestra el IVA agrupado por %:

  IMP.   %   BASE     CUOTA
  IVA   15%  10.00    1.50
  IVA   0%    5.00    0.00

7.2. Arqueo de Caja

Al cerrar la caja (arqueo), el sistema calcula:

  • Total ventas: Suma de totales de ventas del período
  • Desglose de IVA: Porcentaje y monto por cada tasa de IVA
  • Efectivo esperado: Fondo inicial + ventas en efectivo - cambios
  • Cierre automático: La apertura de caja se cierra al hacer arqueo

8. Resumen de Correcciones Aplicadas

#ProblemaSolución
1 IVA duplicado (se cobraba de más) Ahora se extrae del precio, no se suma
2 Stock no validado (podía quedar negativo) Se valida stock antes de añadir al ticket y al guardar venta
3 Cambio negativo (recibido menor que total) Validación server-side, cambio mínimo 0
4 IVA limitado a opciones fijas Ahora cualquier % de 0 a 100
5 Error de sintaxis en PRODUCT_STOCKS Corregido espacio entre {{
6 Cashflow ignoraba errores Ahora devuelve error si falla contabilidad
7 Cierre de caja de otro usuario Filtrado por usuario actual
